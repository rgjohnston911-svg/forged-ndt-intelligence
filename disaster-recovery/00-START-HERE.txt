================================================================================
TROUBLESHOOTING
================================================================================

Problem: "Extension uuid-ossp not found"
Solution: This is standard in Supabase. If missing, run:
  CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

Problem: "Foreign key violation during schema creation"
Solution: Ensure the primary key tables are created first.
  All IF NOT EXISTS guards should handle this automatically.

Problem: "RLS policies failing after creation"
Solution: Verify:
  1. auth.users table exists
  2. User roles are properly configured
  3. org_id values in JWT claims

Problem: "Vector indexes failing"
Solution: pgvector extension may not be enabled:
  1. In Supabase console, add pgvector to extensions
  2. Re-run the schema creation
  3. Or skip vector operations if not needed

Problem: Table X references non-existent table Y
Solution: Load all tables in order (the file does this automatically)
  or use the migration sequence documented in SCHEMA-AUDIT.txt
